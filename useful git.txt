깃허브(issue, branch, merge 활용법)

1. Issue
신고, 제안을 해결하기 위한 작업단위
Issue 번호를 commit 메세지에 #(번호)으로 넣으면
깃허브에서 인지함


2. Branch & merge
Conflict 를 대비하여 각자 다른파일을 수정할때 쓰임

브랜치 생성 Git branch 브렌치이름
현재작업할 위치 설정 Git checkout 브랜치이름
Git add, status, commit
현재작업할 위치 설정 Git checkout master
브랜치 병합 Git merge 브랜치 이름
브랜치 삭제 Git branch -d 브랜치이름
브랜치 목록 조회 Git branch


3. 로컬 repo와 원격 repo(깃허브와 같은)를 연결하는
Tracking 은 branch 단위로 이루어짐
